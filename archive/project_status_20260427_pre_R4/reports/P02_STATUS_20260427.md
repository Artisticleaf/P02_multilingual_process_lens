Observed prompt/model lessons:

- DeepSeek and Phi need longer decoding; earlier short negative results were mostly truncation artifacts.
- Qwen3.5 works better without the chat template in this prompt family.
- Some reasoning models restate the instruction and leak placeholder text such as `Final answer: <answer>`, so strict final-answer parsing must ignore prompt placeholders.
- Phi has substantial English-route language mixing; Qwen3-14B also shows non-trivial Chinese-route mixing in short outputs.

### E03 Span Patch Hard

Span-specific residual patching was run on seven models. This tests whether causal margin shifts come only from the final verifier-token position or also from process-text spans.

| Model | Strongest Clean Span/Layer | Strongest Trace Span | Strongest Support/Error Span |
|---|---|---|---|
| qwen3_14b_base | verdict_pos L30 | trace_span L20 | support_error_span L14 |
| qwen3_8b_base | verdict_pos L27 | trace_span L9 | support_error_span L9 |
| deepseek_r1_0528_qwen3_8b | verdict_pos L27 | trace_span L5 | support_error_span L9 |
| qwen35_9b | verdict_pos L31 | trace_span L3 | support_error_span L3 |
| ministral3_8b_reasoning | verdict_pos L33 | trace_span L7 | support_error_span L7 |
| phi4_mini_reasoning | verdict_pos L31 | trace_span L8 | support_error_span L8 |
| glm46v_flash | trace_span L10 | trace_span L10 | support_error_span L10 |

Interpretation:

- Verdict-position patching is still strongest for most models, so verifier-output priors remain a serious confound.
- However, trace-span and support/error-span patching create clean bidirectional effects in Qwen3, DeepSeek-Qwen, Qwen3.5, Ministral, and Phi. This is the first concrete evidence that process text spans carry causally usable process-validity information before the final verdict token.
- GLM remains weak; it should be treated as a negative or architecture-mismatch control rather than evidence against the hypothesis.
