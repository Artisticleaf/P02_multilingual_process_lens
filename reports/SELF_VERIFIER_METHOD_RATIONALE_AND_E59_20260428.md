# Self-Verifier Rationale and E59 Cross-Verifier Audit / Self-verifier 方法合理性与 E59 跨 verifier 审计

Date / 日期: 2026-04-28 CST

Purpose / 目的：补充说明“同一个模型自己判断自己的 trace”为什么是一个合理的研究方法，同时明确它的偏差风险，并用现有 P0 官方结果做一个不新增推理成本的 E59a cross-verifier 分析。结论先行：self-verifier 是合理且必要的研究对象，但不能单独作为最终可信性结论；它应与 cross-verifier、blind source、order-balanced sibling comparison 和 human audit 共同使用。

## 1. Why self-verification is a reasonable research method / 为什么 self-verifier 合理

Plain-language explanation / 说人话解释：

“同一个模型自己判断自己的 trace”不是奇怪设定。真实系统里经常会让模型先生成，再让同一个模型或同家族模型检查、打分、改写、筛选：例如 self-refine、自我检查、best-of-N 选择、LLM-as-a-judge、RLAIF/AI feedback、合成数据过滤。我们研究 self-verifier，就是在审计这种现实 pipeline 的薄弱环节：如果同一个强模型已经有过程错误信号，但在 absolute Yes/No 自审时仍然放过错误 trace，那么使用同模型/同家族模型做数据筛选或 trace selection 就有风险。

Scientific rationale / 科学理由：

1. Ecological validity / 生态有效性：self-verification/self-evaluation 是实际 LLM pipeline 中常见组件。已有工作把同一模型用于验证候选答案、检查幻觉或作为 judge，因此 self-verifier 是现实系统风险，而不是人为稻草人。
2. Mechanistic attribution / 机制归因：使用同一 checkpoint 做 generator/verifier 或 verifier/mechanism probe，可以回答“模型自己内部是否有过程证据，却没有在输出头/阈值上使用好”。如果只用外部 verifier，就更难归因到被研究模型的 hidden states。
3. Objective diagnosis / 目标诊断：同一个模型在 absolute Yes/No 下失败、在 sibling comparison 下成功，说明问题不是“完全没有能力”，而是 objective/threshold 改变了模型是否使用过程信号。
4. Controlled stress test / 受控压力测试：self-verifier 不是为了证明所有部署都这么做，而是一个严苛且可解释的压力测试；如果 self-verifier 都不稳定，就更需要外部或对比式审计。

## 2. Known risks of self-verifier / self-verifier 的已知风险

Self-verifier 合理，但有明显偏差风险，不能单独使用：

- Self-preference / 自偏好：LLM judge 可能偏好自己或同家族模型写出的答案。
- Self-recognition / 自识别：模型可能能从风格识别自己的输出，导致评估不是纯粹过程判断。
- Position/verbosity/style bias / 位置、冗长、风格偏差：LLM-as-a-judge 已被证明会受 A/B 顺序、回答长度、熟悉风格影响。
- Intrinsic self-correction limit / 内生自纠错限制：没有外部反馈时，模型自我纠错在 reasoning 任务上并不稳定，甚至可能变差。

Therefore / 因此：我们的论文方法应把 self-verifier 定义为“被审计对象”，而不是“可信裁判本身”。最终证据必须依赖 manual labels、order-balanced sibling comparison、cross-verifier replication、hidden-state intervention 和数据泄露审计。

## 3. Literature basis / 文献依据

Self-verification is a recognized paradigm / self-verification 是已有范式：

- Weng et al. (EMNLP Findings 2023) propose self-verification for LLM reasoning and show it can improve arithmetic, commonsense, and logical reasoning by verifying the model's own derived conclusions. / Weng 等提出 self-verification，并显示它能提升若干推理任务。
- SelfCheckGPT (EMNLP 2023) uses a black-box model's own sampled responses to detect hallucinations without external databases. / SelfCheckGPT 使用模型自身采样一致性做无资源幻觉检测。
- Chain-of-Verification (ACL Findings 2024) asks the model to draft, generate verification questions, answer them independently, and revise, showing that structured self-checking can reduce hallucination. / CoVe 让模型生成验证问题并独立回答以减少幻觉。
- Judging LLM-as-a-Judge (NeurIPS 2023 Datasets & Benchmarks) shows strong LLM judges can approximate human preferences at scale, but also documents biases. / MT-Bench/Chatbot Arena 证明 LLM-as-a-judge 可扩展，但也有偏差。

Self-verification has known limitations / self-verification 有已知局限：

- Large Language Models are not Fair Evaluators (ACL 2024) documents positional bias and proposes balanced-position calibration. / LLM judge 会有位置偏差，需要校准。
- LLM Evaluators Recognize and Favor Their Own Generations (2024) reports self-recognition and self-preference risks when a model evaluates its own generations. / 模型可能识别并偏好自己的输出。
- Large Language Models Cannot Self-Correct Reasoning Yet (ICLR 2024) warns that intrinsic self-correction without external feedback is unreliable for reasoning. / 没有外部反馈的自纠错在推理任务上不可靠。

How our work differs / 我们的区别：这些工作主要研究 self-verification 是否能提升答案或评价质量；我们研究的是 self-verifier 在 trace-selection 中是否会因为 final-answer anchoring 与 absolute Yes/No objective 而过度接受 process-invalid trace，并进一步用 sibling comparison 与 residual steering 证明“过程信号存在但没被用好”。

## 5. E59b full mutual-verifier design / E59b 完整互为 verifier 设计

Goal / 目标：真正测试“模型 A 生成的 trace，由模型 A 自己审 vs 模型 B/C 审”是否不同。

Design / 设计：

1. Generation pool / 生成池：让每个 P0 模型在 no-leak prompts 下生成 traces；不提供 known error span，不提供 gold answer，或在 answer-anchor diagnostic 中明确标注不作为自然发生率。
2. Human audit / 人工审计：只对 final-correct traces 做逐句 process-validity 审计，标记 valid, invalid, ambiguous。ambiguous 不进主表。
3. Source-blind verifier / 来源盲审：verifier prompt 不告诉 trace 来自哪个模型，避免显式 source bias。
4. Matrix / 矩阵：source model × verifier model，分别记录 absolute Yes/No false-accept、valid-accept、sibling accuracy、source-family effect。
5. Bias controls / 偏差控制：A/B 顺序平衡、trace length matching、style rewrite/obfuscation、同题同答案 sibling pairs、bootstrap CI。
6. Mechanism extension / 机制扩展：对 self-verifier 与 external-verifier 的 residual evidence 做对比，看 self-preference 是否改变 hidden direction 或输出阈值。

Expected high-value outcomes / 期望信息收益：

- If self-verifier accepts its own invalid traces more than external verifiers, we have direct self-preference/process-risk evidence. / 如果自审更容易接受自己的 invalid trace，就有直接自偏好风险证据。
- If all verifiers over-accept similarly, the issue is broader objective/threshold mismatch, not self-preference. / 如果所有 verifier 都类似过度接受，则问题更像普遍 objective/threshold 错配。
- If sibling comparison fixes both self and cross settings, it strengthens the claim that contrastive objective exposes underused process evidence. / 如果 sibling 同时修复 self/cross 场景，则更强支持“对比目标暴露未充分使用的过程证据”。

## 6. E59c style-controlled mutual proxy / E59c 风格受控互审代理实验

Because naturally generated ACPI is rare in current E48, a useful bridge is style-controlled trace rewriting. / 由于当前 E48 自然 ACPI 稀少，可以先做风格受控改写作为桥接实验。

Design / 设计：take audited controlled valid/invalid traces, ask each P0 model to paraphrase them without changing math facts, then manually audit that process validity is preserved. Run the same source × verifier matrix. / 设计：拿已审定的受控 valid/invalid trace，让每个 P0 模型在不改变数学事实的情况下改写，再人工确认 process validity 保持，之后做 source × verifier 矩阵。

Why useful / 为什么有用：it introduces model-specific surface style while preserving process labels, allowing a cleaner test of self-style preference before full natural ACPI harvesting. / 它引入模型特定表面风格，同时保留过程标签；在自然 ACPI 采集前，可更干净地测试 self-style preference。

Risk / 风险：rewriting itself may repair or worsen the process. Therefore every rewritten row must be re-audited and ambiguous rows excluded. / 风险：改写可能修复或破坏过程，因此每行必须重新人审，ambiguous 排除。

## 7. How to phrase this in the paper / 论文中怎么写

Recommended wording / 推荐表述：

We use self-verification not as an assumption that a model is a reliable judge of itself, but as an object of audit. Self-verification is a common deployment pattern in LLM pipelines, and using the same checkpoint also lets us ask whether process-validity evidence is internally available but underused by the model's own decision objective. To control known LLM-judge biases, we complement self-verification with source-blind cross-verifier analyses, order-balanced sibling comparison, manual process labels, and residual-state interventions.

中文翻译：我们使用 self-verification，并不是假设模型能可靠裁判自己，而是把它作为被审计对象。Self-verification 是 LLM pipeline 中常见部署模式；使用同一个 checkpoint 也让我们能检验“过程有效性证据是否在模型内部可得，但没有被模型自己的决策目标充分使用”。为控制 LLM judge 的已知偏差，我们同时使用来源盲化的 cross-verifier 分析、顺序平衡的 sibling comparison、人工过程标签和 residual-state 干预。
